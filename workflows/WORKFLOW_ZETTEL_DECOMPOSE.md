# 제텔카스텐 분해 워크플로우

## 개요
HWP/HWPX 텍스트 → 최대 세분화 제텔카스텐 노트 분해

## 입력
- `source_file`: 텍스트 파일 경로 (inbox/기능분석_텍스트/XXX.txt 등)
- `output_dir`: ideas/ 하위 카테고리 폴더
- `id_start`: 시작 ID (YYYYMMDDHHNN 형식)
- `id_end`: 종료 ID (범위 초과 금지)
- `instruction_file`: 분해 지시문 (있으면)

## 실행 단계

### 1단계: 원문 읽기
- `source_file` 전체 읽기
- 기관명/문서명 확인

### 2단계: 구조 파악
- 목차/장절 구조 파악
- 분해 단위 결정 (원자적 — 하나의 아이디어/사실/정책 = 하나의 노트)

### 3단계: 노트 생성
- 각 노트: `{id}_{제목}.md`
- 형식:
```markdown
# {제목}
ID: {YYYYMMDDHHNN}
Tags: #카테고리 #세부주제

{본문 — 원문의 핵심 내용을 압축·재구성}

## 링크
- [[관련노트ID]] 관련 설명
```

### 4단계: 인덱스 업데이트
- `{output_dir}/000_{카테고리}_INDEX.md` 생성/업데이트
- 전체 노트 목록 + 소분류

## 제약사항
- ❌ ID 범위 초과 금지 (다른 sub-agent와 충돌)
- ❌ 기존 노트 덮어쓰기 금지
- ✅ 최대 세분화 (서호성님 선호)
- ✅ 양방향 링크 필수
- ✅ 같은 카테고리 내 + 다른 카테고리 간 교차 링크 권장

## 완료 조건
- 모든 원문 내용이 노트로 분해됨
- 인덱스 파일 생성/업데이트됨
- 노트 수 보고
